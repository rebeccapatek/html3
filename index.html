Tables

To display data in a table format, we must create the table to content the data using the <table> element.

In html, the rows, columns, and cells that will hold data must be created individually.

The first step in entering data into the table is to add rows using the table row element.

<tr></tr> for each one

Adding rows won't actually add any cells with data to your table. Each cell element must also be defined. In order to do this you must add a table data element. For each row, add as many table data elements as you would like for columns across.

To add titles to rows and columns, you can use the table heading element: <th>. These elements helps to describe what the data within the table represents. Just like table data, a table heading element must be placed within a table row.

This code:

<table>
   <tr>
        <th></th>
        <th scope="col">Saturday</th>
        <th scope="col">Sunday</th>
    </tr>
    <tr>
        <th scope="row">Temperature</th>
        <td>73</td>
        <td>81</td>
    </tr>
</table>
Would display this:
Saturday Sunday
Temperature 73 81

Note that the first row has a blank heading, and a Saturday, Sunday heading. The blank heading creates the extra table cell necessary to align the able headings correctly over the data that matches up with it. In the second row, the table heading Temperature was added.

Note also the use of the scope attributes.

  1. row - this value makes it clear that the heading is for a row.
  2. col - this value makes it clear that the heading is for a column.

In order to make a table more readable, you should add a border. The convention now is top do this in CSS. For example:

table, td {
  border: 1px solid black;
}

would be stored in a style.css document.

Another attribute of the <td> is the colspan. This attribute accepts an integer greater or equal to one to denote the number of columns it spans across.

You could also assign the rowspan attribute to a <td> with a value greater or equal to the number or rows that you want the data to cover.

For long tables, you can use the table body element <tbody> to section off all of the table's data, and exclude the table's headings. It makes sense to section off the table's column headings using the table head <thead> element as well when you are sectioning off the table' body.

The Table footer element (<tfoot>) is another way to section off a longer table. Note that the footer element must still contain the table row element and any table data within that row. Table footers are ofter used to contain sums, differences, and other data results.

Tables by default are very blank. They have no borders, the font color is black, and the typeface is the same type used for other HTML elements.

CSS (Cascading Style Sheets) is a language that web developers use to style the HTML content on a web page. You can use CSS to style tables. Specifically, it is nice to style the various aspects mentioned above. Here is an example of the styling you might have for a table:

table, th, td {
  border: 1px solid black;
  font-family: Arial, sans-serif;
  text-align: center;
}

How Does the INTERNET work?

You can think of the internet as a network of computers which send and receive information. Computers need an HTTP request to know how to communicate. The HTTP request instructs the receiving computer how to handle the incoming info. HTTP stands for Hypertext Transfer Protocol and is used to structure requests and responses over the internet. HTTP requires data to be transferred from one point to another over the network. Think of HTTP command language that the devices on both sides of the connection must follow in order to communicate.

The transfer of resources happens using TCP (Transmission Control Protocol). In viewing this webpage, TCP manages the channels between your browser and the server. TCP is used to manage many types of internet connections in which one computer or device wants to send something to another.

HTTP & TCP: How it Works

When you type an address into your browser, you are commanding it to open a TCP channel to the server that responds to that URL (or Uniform Resource Locator). A URL is like your home address or phone number because it describes how to reach you.

In this situation, your computer, which is making the request, is called the client. The URL you are requesting is the address that belongs to the server.

Once the TCP connection is established, the client sends a HTTP GET request to the server to retrieve the webpage it should display. After the server has sent the response, it closes the TCP connection. If you open the website in your browser again, or if your browser automatically requests something from the server, a new connection is opened which follows the same process described above. GET requests are one kind of HTTP method a client can call.

Let’s explore an example of how GET requests (the most common type of request) are used to help your computer (the client) access resources on the web.

Suppose you want to check out the latest course offerings from http://codecademy.com. After you type the URL into your browser, your browser will extract the http part and recognize that it is the name of the network protocol to use. Then, it takes the domain name from the URL, in this case “codecademy.com”, and asks the internet Domain Name Server to return an Internet Protocol (IP) address.

Now the client knows the destination’s IP address. It then opens a connection to the server at that address, using the http protocol as specified. It will initiate a GET request to the server which contains the IP address of the host and optionally a data payload. The GET request contains the following text:

GET / HTTP/1.1

Host: www.codecademy.com

What is HTTPS?

Since your HTTP request can be read by anyone at certain network junctures, it might not be a good idea to deliver information such as your credit card or password using this protocol. Fortunately, many servers support HTTPS, short for HTTP Secure, which allows you to encrypt data that you send and receive.

HTTPS is important to use when passing sensitive or personal information to and from websites. However, it is up to the businesses maintaining the servers to set it up. In order to support HTTPS, the business must apply for a certificate from a Certificate Authority.

Forms

An HTML form element (<form<) is responsible for collecting information to send somewhere else.

Example:

<form action="/example.html" method="POST"> </form>

In the above example, we’ve created the skeleton for a <form> that will send information to example.html as a POST request:

The action attribute determines where the information is sent.

The method attribute is assigned a HTTP verb that is included in the HTTP request.

Let's first explore a text value for the type attribute.

<form action="/example.html" method="POST"> <input type="text name="first-text-field">

It is also important that we include a name attribute otherwise the information typed in the input field won't be sent when the form is submitted.

When initially loaded it would be an empty box.

Let's say the user typed the words, "important details" into the text box. Then when the form is submitted:

"first-text-field"=important details" would be sent to /example.html because the value of the name attribute is "first-text-field" and the value of value is "important details"

The label element is so that the user can identify what the text field is for. It will display what text is written between the opening and closing tags. To associate a label element with its corresponding input field, the input needs an id attribute and the label needs a matching for attribute.

<label for="meal">What do you want to eat?</label> <br> <input type="text" name="food" id="meal">

for an input type of number will restrict what users can type into the input field. Just numbers and +, -, and .

A step attribute equal to 1 creates arrows inside the input field to increase or decrease the number by 1.

Let's explore the Checkbox Input

If we presented multiple options to users and allow them to select any number of options then we would need a checkbox for the type attribute of an input element.

You need to create labels for each of the checkboxes, otherwise the user would just see boxes.

The id attribute of the checkbox is what pairs it with the label. The name attribute is what gives it a value. The value attribute is what the value is set equal to. So the name attribute could be the same for several different checkboxes. Have the same name attribute will group these inputs together.

The Radio Input.

Checkboxes work well if we want to present users with multiple options and let them choose one or more of the options. However, there are cases where we want to present multiple options and only allow for one selection — in these cases we need to use the radio input option.

Very much like the checkboxes, we need to also include label elements for the user to know what they are choosing.

Radio buttons that are grouped together under one question should have the same name.

The value would be what differentiates them.

Dropdown lists

Allows our users to choose one option from an organized list.

You would use a select element (opening and closing tags) and within the select element you would have option elements (with closing tags) for each of the items you'd like displayed in the dropdown.

Each option has a value attribute. The value is set equal to the name of the select.

Datalists

They are nested within an input type of text. The inputs with type of text also needs to have a list attribute whose value matches the id attribute of the datalist. Users can type into the text field and filter options from the datalist. If non of the options match what the user types, the user can still submit what they've typed in. When the form is submitted, the value that the user typed in or selected is submitted paired with the name of the input field.

You need to have an option element nested under the datalist element. The value of these options are what will display when you are searching
